1. setup.sh
SCTで変換エラーを発生させるオブジェクトを作成します。環境作成中に複数のDBユーザーを作成し、オブジェクトを作成します。本番環境などでは実施しないでください。

ADMIN_USERNAME: RDSのマスターユーザー名などDBAユーザー名を指定してください
ADMIN_PASSWORD: 上記ユーザーのパスワードを指定してください
DB_ENDPOINT:    RDSの場合はエンドポイント、それ以外はホスト名を指定してください
DB_PORT:        リスナーのポートを指定してください
DB_NAME:        データベース名を指定してください

以下は、環境作成中に利用する新規ユーザーの情報になりますが、デフォルトで問題ありません。
DB_USERNAME:      基本的に本スクリプトではここで指定されたユーザーを所有者としてオブジェクトを作成します
DB_PASSWORD:      上記ユーザーのパスワード
DB_0032_USERNAME: 複雑な所有者の状況を再現させるためのユーザーになります。これは0032.sqlないで実行されます。
DB_0032_PASSWORD: 上記ユーザーのパスワード

2. cleanup.sh
setup.shで作成したオブジェクトをユーザーを含めて削除します。
